## Key Fixes

### Bug: `_map_results` returned 0 fields
**Root cause**: LLM (gemma3:1b) returns `{"field_key": "value", ...}` (key as JSON key) instead of `{"key": "field_key", "value": ...}`. Fixed `_map_results` to detect field keys in both positions.

### Bug: JSON parsing failures with minicpm-v
**Root cause**: minicpm-v returns invalid JSON patterns:
1. Wraps arrays in `{}` instead of `[]`: `"{ {...}, {...} }"`
2. JavaScript-style comments: `// Note: ...`
3. Unquoted values: `490 MPa` instead of `"490 MPa"`

**Fix**: 
- Balanced-brace JSON extraction fallback
- JS comment stripping with regex
- Handle `{...}` wrapped arrays

### Feature: minicpm-v vision model support
- Added `OLLAMA_VISION_MODEL=minicpm-v` to `.env`
- `_call_ollama` now handles vision models via `/api/generate` with `images=[b64]`
- `_supports_vision()` checks `OLLAMA_VISION_MODEL` env var
